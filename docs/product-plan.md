# 제품 계획

## 현재 목표

PDF에 들어 있던 DOOH 매체 정보를 광고주가 바로 탐색할 수 있는 정적 카탈로그로 재구성한다.

## 정적 HTML로 시작하는 이유

- PDF 기반 원천 데이터의 정확도 검수가 먼저 필요하다.
- 가격과 구좌 가능 여부는 상담 시점에 변동된다.
- DB와 인증 없이도 매체 탐색, 상세 확인, 견적 문의까지 충분히 제공할 수 있다.
- JSON/CSV로 분리하면 추후 DB 이전이 쉽다.

## 향후 플랫폼화에 필요한 기능

- 매체 데이터 관리 화면
- 가격 변경 이력
- 구좌 가능 여부 관리
- 문의 내역 저장
- 제안서 자동 생성
- 집행 리포트 공유
