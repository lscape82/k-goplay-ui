## A. SEO / GEO / AEO — 개발자 전달 사항

### A-0. 대원칙 (판단 기준)
- ⚠ **지도(map.html)의 카드·버튼·요약·필터는 전부 JS 렌더 → 검색봇이 못 읽음.** 지도에서 브랜드명을 가리거나 라벨을 바꿔도 **SEO 손실 0**. (순수 UX 영역)
- ⚠ 반대로 **봇이 읽는 정적 HTML 3곳은 브랜드명·검색키워드를 반드시 유지**:
  1) 엘리베이터 상세페이지(elevator-*.html) 2) 옥외광고 매체목록(media-catalog.html) 3) llms.txt
  - 예: "타운보드광고"·"KT타운보드"·"포커스미디어" 등 등록 키워드를 이 페이지들이 받아야 함. → 상세페이지의 "타운보드"를 "T사"로 바꾸면 검색노출을 잃음. **금지.**

### A-1. 엘리베이터 상품 상세페이지 (신설 완료 — 봇 노출 대상)
- ☑ 상세 7장 + 유형 허브 2장:
  - elevator.html (상품 6개 비교 허브)
  - elevator-townboard / -focus / -mediamid / -officebiz / -prime-living / -prime-office.html
  - elevator-apartment.html · elevator-office.html (유형별 패키지 안내)
- ▢ 각 페이지 스키마 최종 점검: `Product` / `Offer` / `FAQPage` (규모·대당 단가범위·계산예시·지역분포·단가표(타운보드 21행)·FAQ)
- ▢ 브랜드 키워드 본문 반영 확인(타운보드/포커스/미디어믿/오피스비즈TV/프라임리빙/프라임오피스)

### A-2. 내부 링크 (사이트 안에서 상세페이지로 가는 길)
- ☑ 지도 카드 "상세보기 ›" → `ELEV_DETAIL_PAGE[networkId]` (elevator-*.html). *(이전엔 지도→상세 내부링크 0건이었음, 해소)*
- ☑ 지도 세그먼트 아래 "○○ 패키지 상품 보기 →" → elevator-apartment / elevator-office.html
- ▢ **정적 페이지 간 내부링크 추가 필요**: 헤더/푸터/매체목록 등 봇이 읽는 곳에서 elevator 페이지로 가는 링크. (지도 링크는 JS라 봇이 못 봄 → 정적 링크가 별도로 있어야 SEO 효과)
- ▢ elevator.html 허브를 헤더 or 옥외광고 매체 메뉴에서 연결할지 결정 🅐

### A-3. sitemap.xml
- ▢ 신규 elevator-*.html 9장이 sitemap에 등록됐는지 확인 (기존 532개)
- ▢ lastmod 갱신

### A-4. llms.txt (AEO/GEO — LLM 답변 노출용)
- ☑ 파일 존재 (7.2KB, 2026-07-17)
- ▢ 엘리베이터 상품 6종·규모·단가 요약이 llms.txt에 반영됐는지 확인
- ▢ 상품별 상세페이지 URL이 llms.txt에 링크됐는지 확인

### A-5. GEO(생성형 검색) 공통
- ▢ 각 상세페이지 제목/메타디스크립션에 "지역 + 매체유형 + 상품" 조합 키워드
- ▢ FAQ를 질문형("아파트 엘리베이터 광고 단가는?")으로 → AI 답변 인용 유도

## B. 라이브 페이지 적용 시 주의사항 (배포 전 필독)

### B-1. 배포 규칙
- ⚠ **요청 전 절대 배포 금지.** 모든 수정은 샘플(localhost)에 먼저 → 확인 후 "배포해줘" 지시 시에만.
- 배포 = `main` push → GitHub Pages 자동 반영 (필요 시 수동 rebuild 트리거).
- 샘플링크 = localhost:3000 (로컬 프리뷰) · 배포링크 = https://lscape82.github.io/k-goplay-ui/

### B-2. 캐시 버스팅 (안 하면 옛 화면이 뜸)
- ⚠ **map.html의 `?v=` 버전 문자열을 수정 때마다 bump 필수.** 안 바꾸면 브라우저가 옛 map.js/CSS를 서빙 → 변경이 안 보임(강력새로고침 필요). 이번 세션에서 계속 bump해 옴.
- 대상: `map.js?v=…`, `map-services.css?v=…`, `styles.css?v=…`

### B-3. 원본 데이터 위치 (저장소에 없음)
- ⚠ 엘리베이터 원본 엑셀은 **`E:\2. 광고플레이(주)\…\2026 UI활용데이터\엘리베이터\`** 폴더. (Desktop 아님 — 과거 오판 있었음)
- 재생성 시 `build-elevator-sites.py` / `build-elevator-data.py`의 SRC 기본값이 이 경로.

### B-4. 프로토타입 vs 라이브 (건드리면 안 되는 것)
- ⚠ 큐레이션 테마 미연동·매체 사진 샘플 돌려쓰기는 **버그 아님**(라이브는 서버 데이터). 결함으로 보고/수정 금지.
- ⚠ `media.json` 목록 순서는 **수작업 큐레이션** — 빌드 스크립트로 통째 재생성 금지, 필드만 in-place 패치.

### B-5. 동시 편집 충돌
- ⚠ 여러 세션이 map.html / map.js 동시 편집 시 파일이 도중에 바뀜. 편집 전 재확인, 가능하면 한 번에 한 세션만.

### B-6. 엘리베이터 데이터 상태 (라이브 반영 전 확인)
- 사이트 총 ~10,075곳 지오코딩(표본). 상품별: 타운보드 3,650 · 포커스 4,707 · 미디어믿 869 · 오피스비즈TV 175 · 프라임오피스 606 · 프라임리빙 68.
- ⚠ **프라임오피스 141건은 어디에도 주소가 없어 핀 없음**(606/747). 네트워크 집계(747·4,847대)에는 잡히므로 요약·상세 수치는 747 기준 정확. → 패키지 전용이라 개별 핀 없어도 무방.
- ▢ 미완: 샘플 사진(assets/images/elevator/*, 현재 임시), 상품 실사진.
